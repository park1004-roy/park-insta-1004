## 1. 제품 한 줄 정의

> Instagram DM으로 들어오는 반복 문의를 키워드 기반으로 자동 감지하여 사전에 등록한 답변을 즉시 발송하고, 운영자가 규칙·로그·테스트를 웹 화면에서 직접 관리할 수 있는 **단독 운영형 DM 자동응답 시스템**

---

## 2. 왜 이 서비스가 필요한가

### 문제 상황

Instagram 비즈니스 계정을 운영하면 DM으로 동일한 유형의 문의가 반복적으로 들어옵니다.

- "가격이 어떻게 되나요?"
- "배송은 얼마나 걸려요?"
- "링크 보내주세요"

운영자가 이 메시지에 하나하나 답변하려면:

1. **시간이 반복 소모됩니다** — 같은 답변을 계속 수동으로 입력해야 합니다.
2. **응답 속도가 느려집니다** — 자리를 비운 사이 문의가 누적되면 늦게 답변이 갑니다.
3. **응답이 누락됩니다** — 바쁠 때 일부 DM을 놓치는 일이 생깁니다.
4. **추적이 되지 않습니다** — 어떤 문구에 어떤 답변이 나갔는지 기록이 없습니다.
5. **실험이 불가능합니다** — 새로운 답변 문구를 시험해볼 수 있는 운영 화면이 없습니다.

### 이 서비스가 해결하는 것

| 문제 | 해결 방식 |
|------|----------|
| 반복 답변 노동 | 키워드 규칙으로 자동 발송 |
| 느린 응답 속도 | DM 수신 즉시 자동 처리 |
| 응답 누락 | 24시간 자동 응답 엔진 가동 |
| 추적 불가 | 수신/발송 로그 전체 기록 |
| 운영 실험 불가 | 테스트 매칭 화면에서 미리 검증 |

## 3. 누가 쓰는 서비스인가

### Primary User: 단독 운영자 1명

이 서비스는 **팀 협업이 아닌, 혼자 빠르게 운영하는 구조**로 설계되었습니다.

| 항목 | 내용 |
|------|------|
| 역할 | Instagram 비즈니스 계정 운영자 |
| 사용 목적 | DM 자동응답 규칙 설정 및 운영 결과 확인 |
| 기술 수준 | 웹 브라우저 수준의 조작 가능 |
| 사용 환경 | 자신의 컴퓨터에서 로컬 실행 |
| 로그인 여부 | 없음 (혼자 쓰는 도구) |

### Secondary User

없음. 이 서비스는 **Single Operator Tool**입니다.

## 4. 서비스가 하는 일 한눈에 보기

```
[인스타그램 사용자가 DM 발송]
            ↓
[Meta(인스타그램 운영사) 서버가 즉시 알림 전송]
            ↓
[자동응답 엔진이 메시지 수신 및 기록]
            ↓
[등록된 규칙과 키워드 대조]
            ↓
        ┌───────────────────────────────────┐
        │ 키워드 일치?                       │
        │                                   │
        ├─ YES → 해당 규칙의 답변 자동 발송  │
        │         발송 결과 기록              │
        │                                   │
        └─ NO  → 기본 안내 메시지 발송 (선택) │
                  미매칭 로그 기록            │
        └───────────────────────────────────┘
            ↓
[관리자 웹 화면에서 모든 이력 확인 가능]
```

운영자가 관리자 화면에서 하는 일:

- 어떤 키워드에 어떤 답변을 보낼지 **규칙 등록**
- 규칙을 켜고 끄는 **활성/비활성 전환**
- 수신된 DM 목록과 매칭 결과 **로그 조회**
- 자동발송 성공/실패 **로그 조회**
- 새 규칙을 등록하기 전 **테스트 문장으로 사전 검증**

## 5. 주요 화면과 기능

관리자 웹앱은 총 7개 화면으로 구성됩니다.

---

### 5.1 대시보드

운영 현황을 한눈에 확인하는 메인 화면입니다.

**표시 항목**

| 항목 | 설명 |
|------|------|
| 총 규칙 수 | 등록된 자동응답 규칙 전체 개수 |
| 활성 규칙 수 | 현재 작동 중인 규칙 개수 |
| 오늘 수신 메시지 | 오늘 들어온 DM 건수 |
| 오늘 발송 성공 | 오늘 자동 발송에 성공한 건수 |
| 최근 수신 메시지 | 최근 들어온 DM 목록 (발신자, 내용, 시간, 매칭 여부) |
| 최근 발송 실패 | 최근 발송에 실패한 건수 및 원인 |

---

### 5.2 규칙 목록

등록된 자동응답 규칙 전체를 관리하는 화면입니다.

**기능**

- 규칙명/키워드로 검색
- 활성/비활성 상태 필터
- 리스트에서 바로 활성화·비활성화 토글
- 규칙 클릭 시 상세 화면으로 이동

**표시 컬럼**

| 컬럼 | 내용 |
|------|------|
| 규칙명 | 등록한 규칙의 이름 |
| 매칭 방식 | 포함(contains) 또는 완전 일치(exact) |
| 키워드 수 | 등록된 트리거 키워드 개수 |
| 응답 미리보기 | 자동 발송될 텍스트 앞부분 |
| 활성 상태 | 현재 작동 여부 (토글 버튼) |
| 우선순위 | 여러 규칙이 동시에 해당될 때 적용 순서 |
| 최종 수정일 | 마지막으로 수정한 날짜 |

---

### 5.3 새 규칙 생성

자동응답 규칙을 새로 만드는 폼 화면입니다.

**입력 항목**

| 항목 | 필수 여부 | 설명 |
|------|---------|------|
| 규칙명 | 필수 | 이 규칙을 구별하는 이름 |
| 설명 | 선택 | 규칙의 용도 메모 |
| 매칭 방식 | 필수 | 포함(contains) / 완전 일치(exact) |
| 트리거 키워드 | 최소 1개 | DM에서 이 단어가 있으면 반응 (복수 등록 가능) |
| 응답 텍스트 | 필수 | 자동으로 발송될 답변 내용 |
| 응답 링크 | 선택 | 답변과 함께 보낼 URL |
| 활성 여부 | 선택 | 저장 즉시 작동할지 여부 (기본값: 활성) |
| 우선순위 | 선택 | 숫자가 높을수록 먼저 적용 (기본값: 100) |

**매칭 방식 설명**

- **포함(contains)**: DM 메시지 어딘가에 키워드가 있으면 반응 → 예) "가격 문의드립니다" 에서 "가격" 키워드에 반응
- **완전 일치(exact)**: DM 메시지 전체가 키워드와 정확히 같을 때만 반응 → 예) "가격" 한 단어만 입력된 경우에만 반응

---

### 5.4 규칙 상세 / 수정

기존 규칙의 상세 내용을 확인하고 수정하는 화면입니다.

**기능**

| 기능 | 설명 |
|------|------|
| 규칙 수정 | 모든 항목 수정 후 저장 |
| 활성/비활성 전환 | 버튼 한 번으로 즉시 전환 |
| 규칙 삭제 | 삭제 시 기존 로그는 보존 |
| 최근 매칭 이력 | 이 규칙이 최근에 반응한 DM 목록 표시 |

---

### 5.5 수신 로그

Instagram에서 들어온 DM 전체 이력을 조회하는 화면입니다.

**기능**

- 키워드 검색 (메시지 내용 / 발신자 ID)
- 매칭 여부 필터 (매칭됨 / 매칭 안 됨)
- 로그 클릭 시 원문 전체 및 플랫폼 메시지 ID 확인

**표시 항목**

| 항목 | 설명 |
|------|------|
| 발신자 | Instagram 사용자 ID |
| 메시지 내용 | DM 원문 |
| 수신 시각 | 메시지가 들어온 시간 |
| 매칭 여부 | 규칙에 걸렸는지 여부 |
| 적용 규칙명 | 어떤 규칙이 매칭됐는지 |

> 수신 로그는 운영자가 직접 생성하지 않습니다. DM이 들어올 때마다 자동 기록됩니다.

---

### 5.6 발송 로그

자동응답으로 발송된 메시지 이력을 조회하는 화면입니다.

**기능**

- 발송 상태 필터 (성공 / 실패)
- 규칙명 검색
- 로그 클릭 시 실패 사유, 발송 텍스트, Meta API 응답 확인

**표시 항목**

| 항목 | 설명 |
|------|------|
| 수신자 | 자동응답이 발송된 Instagram 사용자 ID |
| 적용 규칙 | 어떤 규칙으로 발송됐는지 |
| 발송 상태 | 성공 / 실패 |
| 발송 시각 | 자동 발송된 시간 |
| 실패 사유 | 실패인 경우 원인 메시지 |

> 발송 로그도 자동 기록됩니다. 운영자가 직접 입력할 필요 없습니다.

---

### 5.7 테스트 매칭

실제로 DM을 보내지 않고 규칙이 잘 작동하는지 사전 검증하는 화면입니다.

**사용 방법**

1. 임의 문장 입력 (예: "배송은 얼마나 걸리나요?")
2. 현재 활성 규칙들과 대조
3. 결과 즉시 확인

**결과 표시**

| 항목 | 내용 |
|------|------|
| 매칭 여부 | 규칙에 걸렸는지 여부 |
| 매칭 규칙명 | 어떤 규칙이 반응했는지 |
| 반응한 키워드 | 어떤 키워드가 일치했는지 |
| 발송 예정 텍스트 | 실제 DM이었다면 보냈을 답변 |
| 첨부 링크 | 함께 발송됐을 URL |
| 우선순위 결과 | 여러 규칙 중 어떤 순서로 적용됐는지 |

> 테스트는 실제 DM을 보내거나 로그를 생성하지 않습니다. 순수 검증 전용 화면입니다.

## 6. 시스템 동작 원리

### 6.1 전체 구조

이 서비스는 4개의 레이어로 구성됩니다.

```
┌─────────────────────────────────────────────────────┐
│                  관리자 웹 화면                        │
│          (운영자의 컴퓨터에서 로컬 실행)                 │
│  규칙 관리 / 로그 조회 / 테스트 매칭                    │
└─────────────────────┬───────────────────────────────┘
                      │ 데이터 읽기/쓰기
┌─────────────────────▼───────────────────────────────┐
│                  Supabase                            │
│              (클라우드 데이터베이스)                    │
│  규칙 저장 / 수신 로그 / 발송 로그 / 설정               │
└─────────────────────┬───────────────────────────────┘
                      │ 이벤트 처리 / 자동 실행
┌─────────────────────▼───────────────────────────────┐
│              자동응답 엔진 (Edge Function)              │
│                 (클라우드 서버리스)                     │
│  DM 수신 → 규칙 매칭 → 자동 발송 → 로그 저장           │
└─────────────────────┬───────────────────────────────┘
                      │ Webhook + Send API
┌─────────────────────▼───────────────────────────────┐
│              Meta (인스타그램 운영사) API               │
│         Instagram DM 수신 알림 + 발송 요청             │
└─────────────────────────────────────────────────────┘
```

### 6.2 자동응답 처리 흐름 (6단계)

| 단계 | 동작 | 처리 주체 |
|------|------|----------|
| 1 | Instagram 사용자가 비즈니스 계정으로 DM 발송 | 사용자 |
| 2 | Meta 서버가 자동응답 엔진에 즉시 알림 전송 | Meta API |
| 3 | 수신 메시지 기록 (중복 메시지 자동 차단) | 자동응답 엔진 |
| 4 | 활성 규칙과 키워드 대조 (우선순위 순) | 자동응답 엔진 |
| 5 | 매칭 규칙의 답변 텍스트를 Instagram으로 자동 발송 | 자동응답 엔진 |
| 6 | 발송 결과(성공/실패) 기록 | 자동응답 엔진 |

### 6.3 규칙 매칭 로직

여러 규칙이 동시에 해당될 때는 아래 순서로 처리됩니다.

1. 활성(ON) 상태인 규칙만 대상으로 함
2. 우선순위(숫자) 높은 순으로 정렬
3. 첫 번째로 일치하는 규칙 하나만 적용
4. 매칭 규칙 없으면 → 기본 안내 메시지 발송 (설정된 경우)

### 6.4 중복 메시지 방지

같은 메시지가 여러 번 수신되더라도 중복 처리되지 않도록 Meta 플랫폼 고유 메시지 ID로 중복을 자동 차단합니다.

## 7. 데이터 구조 개요

> 이 섹션은 기술적 상세가 아닌, 서비스에서 관리하는 정보의 종류를 설명합니다.

### 7.1 규칙

자동응답의 핵심. 운영자가 직접 등록하고 관리하는 데이터입니다.

| 정보 | 설명 |
|------|------|
| 규칙명 | 이 규칙을 식별하는 이름 |
| 트리거 키워드 | DM에서 감지할 단어/문구 목록 |
| 매칭 방식 | 포함 일치 / 완전 일치 |
| 응답 텍스트 | 자동 발송될 답변 |
| 응답 링크 | 답변에 첨부할 URL (선택) |
| 활성 여부 | 현재 작동 중인지 여부 |
| 우선순위 | 복수 규칙 적용 시 우선 순서 |

### 7.2 수신 메시지 로그

들어온 DM이 자동으로 기록되는 데이터입니다. 운영자가 직접 입력하지 않습니다.

| 정보 | 설명 |
|------|------|
| 발신자 ID | DM을 보낸 Instagram 사용자 식별자 |
| 메시지 원문 | 수신된 DM 전체 내용 |
| 수신 시각 | 메시지가 들어온 시간 |
| 매칭 여부 | 규칙에 걸렸는지 / 걸리지 않았는지 |

## 10. 운영 KPI

서비스 가동 후 아래 지표로 운영 효과를 측정합니다.

### 10.1 핵심 운영 지표

| 지표 | 설명 | 확인 위치 |
|------|------|----------|
| 총 규칙 수 | 등록된 자동응답 규칙 개수 | 대시보드 |
| 활성 규칙 수 | 현재 작동 중인 규칙 개수 | 대시보드 |
| 일별 수신 건수 | 하루에 들어오는 DM 수 | 대시보드 / 수신 로그 |
| 자동응답 성공률 | 수신 대비 자동응답 성공 비율 | 발송 로그 |
| 미매칭 비율 | 규칙에 걸리지 않은 DM 비율 | 수신 로그 |
| 발송 실패율 | 발송 시도 대비 실패 비율 | 발송 로그 |

### 10.2 지표 해석 가이드

- **미매칭 비율이 높다** → 새로운 키워드 패턴이 많다는 뜻. 수신 로그를 보고 규칙 추가 필요
- **발송 실패율이 높다** → Meta API 토큰 만료 또는 권한 문제. 연동 상태 점검 필요
- **활성 규칙이 적다** → 비활성 규칙이 많은 경우. 정리 또는 재활성화 검토

## 13. 제품 원칙

### 운영자 한 명을 위한 도구

로그인이 없고 권한 관리도 없습니다. 혼자 빠르게 운영하는 구조가 핵심입니다. 복잡한 설정보다 즉각적인 실행이 우선입니다.

### 규칙 중심 설계

자동응답의 모든 것은 규칙에서 시작합니다. 규칙을 잘 설계하는 것이 곧 서비스 품질입니다.

### 로그로 운영한다

- 어떤 메시지가 왔는가
- 어떤 규칙이 적용됐는가
- 어떤 답변이 발송됐는가
- 실패 원인은 무엇인가

이 4가지가 항상 추적 가능해야 합니다.

### 실험하고 검증한다

새 규칙을 등록하기 전 테스트 화면에서 반드시 사전 검증합니다. 실제 DM 발송 없이 결과를 미리 확인할 수 있습니다.

### 서버리스 백엔드

별도의 서버를 직접 운영하지 않습니다. Supabase의 클라우드 인프라 위에 자동응답 엔진이 동작하므로 서버 유지보수 부담이 없습니다.
